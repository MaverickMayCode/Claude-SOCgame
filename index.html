LOG ANALYSIS
grep
Search for text patterns inside files. Your first tool for log forensics.
grep 'Failed password' /var/log/auth.log
grep -i 'error' /var/log/syslog | tail -50
grep '203.0.113.42' /var/log/auth.log | wc -l
journalctl
Query the systemd journal. More structured than raw log files.
journalctl -u ssh --since "1 hour ago"
journalctl -p err -n 50
last / lastb
Show recent login history. lastb shows failed logins specifically.
last -n 30
lastb | head -20
NETWORK ANALYSIS
ss
Modern socket statistics — faster and richer than netstat.
ss -tuln # all listeners
ss -tulpn # listeners with process names
ss -tulpn | grep 4444
netstat
Show connections, routing tables, and port listeners.
netstat -tuln
netstat -ntu | awk '{print $5}' | cut -d: -f1 | sort | uniq -c | sort -rn
lsof
List open files — sockets included. Great for tying processes to ports.
lsof -i # all network connections
lsof -i :4444 # what's on port 4444
lsof -p 6631 -i # connections for PID
tcpdump
Capture and inspect network packets in real time.
tcpdump -i eth0 -n port 80
tcpdump -i any host 203.0.113.42 -w capture.pcap
FIREWALL & ACCESS CONTROL
iptables
Linux kernel firewall. Powerful packet filtering and NAT.
iptables -A INPUT -s 203.0.113.42 -j DROP
iptables -A INPUT -p tcp --dport 80 -m connlimit --connlimit-above 50 -j DROP
iptables -A INPUT -p tcp --dport 80 -m limit --limit 100/min -j ACCEPT
iptables -L -n -v # list all rules
ufw
Uncomplicated Firewall — friendlier iptables front-end.
ufw deny from 203.0.113.42
ufw limit ssh
ufw status verbose
fail2ban-client
Manage fail2ban — automatic IP banning after repeated failures.
fail2ban-client status sshd
fail2ban-client set sshd banip 203.0.113.42
PROCESS MANAGEMENT
ps
Snapshot of running processes with owner, CPU, mem, and command.
ps aux
ps aux | grep suspicious_name
ps aux --sort=-%cpu | head -10
kill / pkill
Terminate processes. kill by PID; pkill by name. Signal 9 = force kill.
kill -9 4872
pkill -9 kworker64
top / htop
Live interactive process monitor. Great for spotting CPU hogs.
top -u root
htop
FILE SYSTEM FORENSICS
find
Search the filesystem by name, time, permissions, owner. Indispensable for hunting malware.
find /tmp -name ".*" -type f
find / -newer /tmp/ref -type f 2>/dev/null
find /tmp -name "kworker64" -delete
chattr / chmod
Modify file attributes. chattr +i makes a file immutable — even root can't overwrite it.
chattr +i /etc/passwd
chmod 600 /etc/ssh/sshd_config
USER & SESSION MANAGEMENT
who / w
Show who is logged in right now. w includes what they are running.
who
w
usermod / passwd
Modify user accounts. Lock compromised accounts instantly.
usermod -L username # lock account
passwd -l username # alternative lock
crontab / systemctl
Attackers use cron jobs and services for persistence. Always check after eviction.
crontab -l -u root
crontab -r -u root # remove all cron jobs for user
systemctl list-units --type=service
ls /etc/cron.d/ && ls /etc/cron.daily/
SSH HARDENING
sshd_config
Main SSH daemon config. Tighten it to limit attack surface.
grep MaxAuthTries /etc/ssh/sshd_config
sed -i 's/MaxAuthTries 6/MaxAuthTries 3/' /etc/ssh/sshd_config
systemctl restart sshd
authorized_keys
Lists SSH public keys allowed to login without a password. Attackers love to inject keys here.
cat /root/.ssh/authorized_keys
sed -i '/attacker/d' /root/.ssh/authorized_keys
